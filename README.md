# Bug Reporter for BK

Backlogへのバグ報告・課題コメントをワンクリックで送信できるChrome拡張機能です。

---

## インストール方法

1. [Chrome ウェブストア](https://github.com/togetoge3/bug-reporter_BK/blob/main/PRIVACY_POLICY.md) にアクセスする
2. 「Chromeに追加」をクリックしてインストールする
3. Chromeのツールバーに表示された拡張機能アイコンをクリックし、サイドパネルを開く

---

## 初期設定

1. サイドパネル右上の「設定」をクリックする
2. **スペースURL** に Backlog のスペースドメインを入力する（例: `yourspace.backlog.com`）
3. **APIキー** を入力する（Backlog の個人設定ページから取得）
4. 「接続テスト」で接続を確認する
5. **プロジェクト** と **課題種別** を選択する
6. 「保存する」をクリックする

---

## 使い方

### スクリーンショットを撮る

| ボタン | 内容 |
|--------|------|
| 表示部分 | 現在表示されている画面をキャプチャする |
| フルページ | ページ全体をスクロールしながらキャプチャする |
| 選択範囲 | ドラッグで指定した範囲をキャプチャする |

キャプチャ後は「編集」ボタンで矢印・テキストなどのアノテーションを追加できます。

### モードを選ぶ

- **新規作成** — Backlogに新しい課題を起票する
- **既存課題にコメント** — 既存の課題にコメントを追加する

### 入力項目

- **見出し** — 課題のタイトルまたはコメントの見出し（任意）
- **補足・再現手順** — 詳細説明。`@` を入力するとプロジェクトメンバーへのメンションが使えます
- **コメント先課題**（コメントモード）— 検索またはピン留め課題から選択する
- **課題種別**（新規作成モード）— Backlogの課題種別を選択する
- **親課題**（新規作成モード）— 任意で親課題を設定する
- **担当者** — 担当者を設定する（任意）
- **開始日 / 終了日** — 日付を設定する（任意）

### ピン留め機能

よく使う課題を最大3件ピン留めしておけます。課題を検索して選択した状態で「ピン留め」ボタンを押すと登録されます。

### 環境情報の自動取得

ページURL・ブラウザ情報・ビューポートサイズなどを自動取得し、課題に添付します。「自動取得した環境情報」をクリックして内容を確認できます。

### 送信する

「Backlogへ送信」ボタンをクリックすると投稿が完了します。完了後に表示されるリンクから、作成した課題またはコメントを直接確認できます。
